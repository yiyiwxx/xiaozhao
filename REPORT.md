# 校招进度聚合器 - 合规与限制说明

## 一、合规审查结论

### 数据来源合法性
- ✅ 仅使用牛客网 **公开话题API**（`/api/sparta/subject/newest-content`），无需登录或认证
- ✅ 已验证 robots.txt 未禁止该路径，站点对 API 接口无明确限制声明
- ✅ 请求频率控制在 **2.5秒以上**，远低于人工浏览速度
- ✅ 遇到 403/429/WAF 页面时 **自动停止**并记录
- ✅ 不绕过任何验证码、登录墙或访问控制

### 个人信息保护
- ✅ 作者 ID 已 **哈希处理**（MD5取前8位），不可逆向还原
- ✅ 不保存作者昵称、头像、联系方式等可识别信息
- ✅ 学历信息仅提取 **作者自述** 的内容，不做推断
- ✅ 不跨平台关联身份

### 数据使用
- ✅ 结果以 **公司级聚合** 呈现，不暴露个人求职轨迹
- ✅ 仅用于个人学习研究，不用于商业用途
- ✅ 所有数据按规范标注来源 URL，可追溯

1. **部分话题返回空记录**：阿里、京东、美团、百度、小米等话题在分页视图中返回空数据。原因可能是：

## 五、当前限制与数据缺口

### 技术限制
1. **事件类型覆盖不完整**：部分公司话题 API 返回空数据或无法翻页
2. **公司识别依赖关键词**：对于非知名公司或简称匹配可能失败
3. **日期歧义**：正则提取的日期（如 `3.19`）默认为当前年份，对于跨年帖子可能错误
4. **薪资计算简化**：总包计算为 `base × months / 10`（单位万），未计入签字费/股票
5. **事件去重有局限**：同一 content_id 中相同 event_type 只保留一个，可能丢失多轮信息
6. **岗位识别依赖精确关键词**：`产品经理` 能识别，但 `PM`、`产经` 等变体可能遗漏

### 数据缺口
1. **无法获取详细面经正文**：详情页需要登录才能读取
2. **薪资数据稀疏**（21条）：
   - 部分薪资信息隐含在帖子中，正则未必能全部覆盖
   - 少数帖子真实反映薪资，多数帖子仅有模糊暗示
3. **学历认证数据有限**：
   - `userBrief.educationInfo` 字段在大部分帖子中为空
   - 学校信息主要依赖作者自述，置信度参差不齐
4. **无法验证事件真实性**：无法区分"炫耀贴"和真实进展

### 明确区分：已验证 vs 推测

| 类型 | 说明 | 示例 |
|------|------|------|
| ✅ 已验证 | 正文明确写出公司、阶段、日期 | "3.19 WXG一面" → 腾讯一面 3月19日 |
| ⚠️ 推测 | 事件日期来自发布时间而非正文 | "刚面完腾讯" → 一面日期=发帖日 |
| 🔶 低置信度 | 内容含歧义或为询问帖 | "请问有人知道腾讯面什么吗" |
| ❌ 不纳入 | 无公司名或事件模糊 | "今天面试了"（未提公司名） |

## 六、启动命令

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 完整运行（抓取+解析）
python -m crawler.pipeline

# 3. 仅增量抓取
python -m crawler.fetcher

# 4. 仅解析已有数据
python -c "from crawler.pipeline import Pipeline; p=Pipeline(); p.run_parse(); p.close()"

# 5. 启动Web界面
python app.py

# 6. 每日增量抓取
python scripts/daily_crawl.py

# 7. 导出样例数据
python scripts/export_sample.py

# 8. 运行测试
python -m pytest tests/test_all.py -v
```
